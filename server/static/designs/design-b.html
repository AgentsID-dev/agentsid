The Problem

Agents are acting without identity

AI agents access databases, send emails, call APIs, and make purchases. But there is no standard way to identify them, limit what they can do, or trace their actions back to a human.

91.5%

Static API keys everywhere

The vast majority of MCP servers rely on hardcoded API keys in environment variables. No rotation, no scoping, no audit trail.

All-or-nothing

No per-tool permissions

Current MCP auth is binary: either the agent can call all tools or none. There is no way to allow "read" but deny "delete."

Zero visibility

Can't trace to a human

When an agent makes a purchase or deletes data, most organizations have no way to know which human authorized it.

How It Works

Four lines of code. Full agent auth.

1

Register an agent

Create a unique identity tied to a human user. Set permissions and an optional TTL.

2

Add the middleware

Drop one line into your MCP server. Every tool call is now validated automatically.

3

Enforce permissions

Per-tool rules with wildcards and conditions. Default deny. Permissions only narrow, never expand.

4

Query the audit log

Every action is logged immutably. Filter by agent, tool, or time range. Export for compliance.

import { AgentsID } from '@agentsid/sdk';

const aid = new AgentsID({
  projectKey: 'aid_proj_...',
});

// Register an agent with scoped permissions
const agent = await aid.registerAgent({
  name: 'research-assistant',
  onBehalfOf: 'user_abc',
  permissions: [
    'search_memories',
    'save_memory',
  ],
  ttl: '24h',
});

// Drop-in MCP middleware
server.use(aid.mcpMiddleware());

// Every tool call is now authenticated,
// authorized, and audited automatically.

Features

Everything agents need to operate safely

⚿

Agent Identity

Every agent instance gets a unique, verifiable identity. Not a shared API key. A specific identity for a specific agent, traceable to its creator.

⚙

Per-Tool Permissions

Granular, deny-first permission rules with wildcard patterns and conditional constraints. Allow "save_*" but deny "delete_*" on the same agent.

↗

Delegation Chains

Every action traces back to a human. Multi-hop delegation supported. Permissions narrow at each hop and never expand.

🔍

Audit Log

Immutable, append-only event log. Query by agent, tool, action, or time range. Export as JSON or CSV for compliance.

⏱

Auto-Expiring Tokens

Set a TTL on any agent identity. Tokens expire automatically. Revoke instantly and all associated tokens become invalid.

🔒

MCP Middleware

Drop one line into any MCP server. The middleware validates tokens, checks permissions, and logs actions. No tool changes needed.
