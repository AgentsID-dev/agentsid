Threat Assessment

SEC-ANALYSIS-001 The Agent Identity Crisis

AI agents are proliferating without identity controls. Here is the threat model.

THREAT-001

No Authentication

Agents access tools and APIs without proving who they are. Any process can impersonate any agent.

Critical
THREAT-002

Unbounded Permissions

Every agent has full access to every tool. No principle of least privilege. One compromised agent exposes everything.

Critical
THREAT-003

No Audit Trail

When something goes wrong, there is no record of which agent did what. Forensics are impossible.

High
THREAT-004

Token Sprawl

Long-lived tokens, shared secrets, hardcoded credentials. No rotation, no revocation, no lifecycle management.

High
System Architecture

ARCH-SPEC-001 How AgentsID Works

01

Register

Create a project and register agents with scoped permissions via SDK or CLI.

02

Authenticate

Agents receive signed JWT tokens. Middleware validates identity on every MCP request.

03

Authorize

Permission checks enforce least-privilege. Agents can only access the tools they need.

04

Audit

Every action is logged to a hash-chained audit trail. Tamper-proof and verifiable.

Implementation Reference

IMPL-REF-001 Integration in Minutes

Add identity to your MCP server with a few lines of code.

import { AgentsID } from '@agentsid/sdk';

// Initialize with your project key
const aid = new AgentsID({ apiKey: process.env.AGENTSID_KEY });

// Register an agent with scoped permissions
const { agent, token } = await aid.registerAgent({
  name: 'data-analyst',
  scopes: ['read:database', 'write:reports'],
  metadata: { model: 'claude-sonnet-4-5-20250514' }
});

// Add middleware to your MCP server
server.use(aid.middleware());

// Validate tokens on incoming requests
const identity = await aid.validate(token);
console.log(identity.agent_id);  // agt_8f3k2m9x
console.log(identity.scopes);    // ['read:database', 'write:reports']
from agentsid import AgentsID

# Initialize with your project key
aid = AgentsID(api_key=os.environ["AGENTSID_KEY"])

# Register an agent with scoped permissions
result = aid.register_agent(
    name="data-analyst",
    scopes=["read:database", "write:reports"],
    metadata={"model": "claude-sonnet-4-5-20250514"}
)

# Validate tokens
identity = aid.validate(result.token)
print(identity.agent_id)   # agt_8f3k2m9x
print(identity.scopes)     # ['read:database', 'write:reports']

# Check permissions before tool execution
if aid.check_permission(identity, "read:database"):
    results = db.query("SELECT * FROM analytics")
# Register an agent
curl -X POST https://api.agentsid.dev/v1/agents \
  -H "Authorization: Bearer $AGENTSID_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "data-analyst",
    "scopes": ["read:database", "write:reports"]
  }'

# Response
{
  "agent_id": "agt_8f3k2m9x",
  "token": "aid_eyJhbGciOi...",
  "scopes": ["read:database", "write:reports"]
}

# Validate a token
curl https://api.agentsid.dev/v1/validate \
  -H "Authorization: Bearer aid_eyJhbGciOi..."
Technical Specifications

SPEC-SHEET-001 System Capabilities

AUTH

Agent Authentication

Cryptographic identity for every agent. JWT tokens with configurable TTL and automatic rotation.

ALGO: HS256 / RS256 | TTL: 1h default
PERM

Scoped Permissions

Fine-grained access control. Define exactly which tools and resources each agent can access.

MODEL: RBAC + Scopes | GRANULARITY: per-tool
AUDIT

Hash-Chained Audit

Every action logged to a tamper-proof chain. Cryptographic verification of log integrity.

HASH: SHA-256 | CHAIN: append-only
DLGT

Agent Delegation

Agents can delegate sub-tasks with automatically scoped-down permissions. Full chain of custody.

MAX DEPTH: configurable | SCOPE: inherit-down
SDK

Multi-Language SDKs

TypeScript, Python, Ruby, and Java SDKs. Drop-in middleware for any MCP server implementation.

LANGS: TS, PY, RB, JV | FORMAT: REST + SDK
REVK

Instant Revocation

Revoke any agent or token immediately. Emergency kill-switch for compromised credentials.

LATENCY: <100ms | PROPAGATION: immediate
